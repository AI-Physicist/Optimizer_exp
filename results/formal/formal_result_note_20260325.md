# Formal Experiment Note (2026-03-25)

## 1) 正式实验设置
- 模型：decoder-only Transformer（约 9.80M 参数）
- 数据：synthetic random token（next-token 预测，target 为 roll）
- 训练步数：1000
- 固定学习率：0.001（由短程 sweep 选出）
- seed：42
- 理论下界：`ln(32000) ≈ 10.3735`

## 2) 正式实验排名（按 final loss，越低越好）
1. `rmsprop_pnorm (r=1.0)`：`10.385182`
2. `rmsprop`：`10.410859`
3. `rmsprop_no_memory`：`10.417166`
4. `adafactor`：`10.421537`
5. `sgd`：`10.492547`
6. `adamw`：`10.541507`

结论：最优方法和次优方法之间的差距是小量级，全部方法都在向理论下界靠近。

## 3) 为什么这次实验里“体感差别不大”
这是预期内现象，主要由任务属性决定：

- 任务本身简单：随机 token + roll target，最优可达区域清晰，模型在 1000 steps 内容易进入“接近下界”的区间。
- 数据噪声结构单一：没有复杂语义结构，优化器之间在长期阶段容易收敛到相近性能。
- 统一配置抹平差异：固定了模型、batch、steps、scheduler（无额外技巧），最终比较更像“预条件器细节差异”，而不是“任务难度差异”。
- 指标尺度上接近地板：当 loss 已逼近 `ln(V)` 时，不同优化器的可见收益天然变小。

## 4) 为什么这次 RMSprop（及其变体）能超过 AdamW
这里给出机制层面的解释（不是绝对定律）：

- 当前固定学习率是由 `rmsprop_pnorm` 的 sweep 选出的（`lr=0.001`）。这对 RMSprop 系列更友好，不一定是 AdamW 的最优点。
- 该任务上“按坐标缩放/归一化”收益很直接，RMSprop 家族在这点上有效；而 AdamW 在这个设置下的动量+权重衰减优势没有充分体现。
- synthetic 任务 + 短程训练下，AdamW 常见的泛化/稳定性优势不明显，导致其最终 loss 可能略高于更“直接归一化”的方法。

因此，本结果更应理解为：
- 在**这个任务 + 这个固定 lr**下，RMSprop 家族匹配度更高；
- 不是“RMSprop 普遍优于 AdamW”的普适结论。

## 5) 为什么选择 1-范数（r=1）
选择依据是 sweep 结果而不是先验偏好：

- 在短程 sweep（200 steps）中，`r=1.0` 明显优于 `r=1.5/2/3`，且无发散。
- `r=1` 对应的归一化更接近“按绝对值缩放/近 sign 的更新形态”，在本任务里表现出更快逼近低 loss 区域的趋势。
- 这是经验选择：当前证据支持 `r=1` 是本实验的工作点；是否可泛化到更复杂任务，需要后续多 seed、长步数、真实数据验证。

## 6) 后续建议（和 README 计划一致）
- 多 seed 复验当前 ranking 的稳健性。
- 对 AdamW 做单独 lr 小 sweep（例如 `3e-4, 5e-4, 1e-3`）再做公平对比。
- 在更长训练步数（如 5k/10k）下观察排名是否变化。
